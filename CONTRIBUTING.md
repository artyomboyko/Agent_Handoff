# Contributing / Участие в проекте

Thank you for helping improve Agent Handoff.
Спасибо за помощь в развитии Agent Handoff.

## Language / Язык

Use the language of the repository or the Issue.
Используйте язык репозитория или Issue.

- English repository or Issue: write in English.
- Russian repository or Issue: write in Russian.
- Mixed documentation should keep English and Russian sections aligned.

- Английский репозиторий или Issue: пишите на английском.
- Русский репозиторий или Issue: пишите на русском.
- Двуязычная документация должна сохранять соответствие английской и русской частей.

## Workflow / Процесс

1. Open or select a GitHub Issue. / Создайте или выберите GitHub Issue.
2. Check existing claims and linked PRs. / Проверьте существующие claims и связанные PR.
3. Use a meaningful branch name. / Используйте осмысленное имя ветки.
4. Open a Draft PR early. / Рано откройте Draft PR.
5. Run checks or document why they were not run. / Запустите проверки или объясните, почему они не запускались.
6. Keep `ai/` files compact. / Держите файлы `ai/` компактными.
7. Add a handoff when useful. / Добавьте handoff при необходимости.

## Branch names / Имена веток

Use meaningful names without `/`, Issue numbers, or random identifiers by default.
Используйте осмысленные имена без `/`, номеров Issue и случайных идентификаторов по умолчанию.

Examples / Примеры:

```text
workflow-branch-naming
protocol-work-claim
docs-readme-quickstart
refactor-handoff-index
research-github-flow
```

## Pull requests / Pull Requests

A PR should include:
PR должен включать:

- related Issue / связанное Issue;
- summary / краткое описание;
- checks or smoke-test notes / проверки или smoke-test notes;
- risks / риски;
- state updates when needed / обновления состояния при необходимости.

## Documentation parity / Соответствие документации

When changing a user-facing document, update the matching English or Russian document too.
При изменении пользовательского документа обновляйте соответствующий английский или русский документ.

## Private data / Приватные данные

Do not commit secrets, tokens, private keys, credentials, private logs, or private user data.
Не добавляйте secrets, tokens, private keys, credentials, private logs или private user data.
