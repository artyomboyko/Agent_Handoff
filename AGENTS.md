# Repository Guide

Before making changes in this repository:

1. Read `ai/README.md`.
2. Read `ai/HANDOFF_PROTOCOL.md`.
3. Read `ai/PROJECT_STATE.md`.
4. Read `ai/DECISIONS.md`.
5. Check the related GitHub Issue or Pull Request.
6. Check `git status`, the current branch, and recent commits.
7. Check `ai/handoffs/INDEX.md` for relevant active handoffs.

Use GitHub Issues and Pull Requests as the primary workflow system.

Keep `ai/` files small and compact.

Do not duplicate large diffs, bulky records, GitHub discussions, or GitHub Actions data in `ai/`.

Private access values stay outside repository memory.
