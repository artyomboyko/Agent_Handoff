# Repository Guide

Before making changes in this repository:

1. Read `AGENT_HANDOFF_STANDARD.md`.
2. Read `ai/README.md`.
3. Read `ai/HANDOFF_PROTOCOL.md`.
4. Read `ai/AGENT_IDENTITY.md`.
5. Read `ai/WORK_CLAIM_PROTOCOL.md`.
6. Read `ai/PROJECT_STATE.md`.
7. Read `ai/DECISIONS.md`.
8. Check the related GitHub Issue or Pull Request.
9. Check `git status`, the current branch, and recent commits.
10. Check `ai/handoffs/INDEX.md` for relevant active handoffs.

Use GitHub Issues and Pull Requests as the primary workflow system.

Keep `ai/` files small and compact.

Do not duplicate large diffs, bulky records, GitHub discussions, or GitHub Actions data in `ai/`.

Private access values stay outside repository memory.
